### Full report

STATISTICS/METHODS AUDIT — RADiCAL 2023 estimator implementations (read-only; all claims verified from source with file:line; one Monte Carlo run in scratchpad).

== 1. tebSigma (lib/physics/RadTiming.h:27-61) ==
Algorithm: (a) 5σ-free robust seed — iterative 2.5σ-truncated mean/RMS, 5 iterations (lines 33-36); (b) debias rw/0.9546 (line 37); (c) 120-bin histogram over rc±4rw, FitGaussCore ±2σ two-pass (lines 46-49); (d) skew/kurt tail guard |sk|>3 or ku>20 → fall back to robust (lines 54-56); (e) accept the fit only within [0.5,2]× the robust width (line 59). Verdict: statistically sound and unusually well documented — the window is built from a robust statistic precisely because the 2026-06 monotonicity audit showed the old 5σ-RMS window was sculpted by ~30σ outliers (comment, lines 39-45). The guard thresholds are catastrophe switches, not tuning knobs: SE(kurtosis)≈√(24/1000)=0.15, so ku>20 cannot trip on clean data. Two defects: (i) VERIFIED BIAS — 0.9546 is exact for a fixed ±2.5σ cut, but the iteration cuts at 2.5×(truncated RMS); the fixed point is ±2.34σ where the debias should be ~0.938. Monte Carlo (20 trials × 200k standard normals, the exact 5-iteration loop): robust/σ_true = 0.9805±0.0006 (0.982 with 1% 30σ outliers) — the fallback width runs 2.0% LOW. It does not touch the headline (the Gaussian-core fit path is used when the guard passes) but the comment 'never garbage' overstates; at 25.7 ps, 2% is 0.5 ps, comparable to the quoted ±0.6 stat error, so if any published point ever takes the fallback path this matters. (ii) The claim 'clean cores have |skew|<~1, kurt<~15' (line 53) is asserted from experience with no committed measurement. Edge cases handled: n<50 → −1; degenerate window floored at 0.04 ns (line 46); n<20 truncation collapse breaks the loop (line 36); sub-floor guard s>12 ps in both callers (lines 142, 173) is a hard physics prior, documented at line 139-141.

== 2. Gaussian-core fit + sigma-error conventions ==
FitGaussCore (lib/viz/PlotUtils.h:105-145): mean/RMS-seeded two-pass ±nsig gaus, 'RQN' (Neyman χ² on the binned hist; fine at N≥400 core populations), sigma from |par2|, errors = Minuit parabolic. Convention audit across uses: point errors on σ_t are σ/√(2N) EVERYWHERE in the gates (timingFitSummary.C:57, systematicsPostfix.C:114, methodGainPostfix.C:109, all with K=1000 → √2000) and in ~15 study macros (grep: floorErr.C:73, elbowFractionTrend.C:114, uniformityScan.C:306, …) — a real house convention, correct for a full-sample Gaussian σ and mildly optimistic (~10-20%) for a ±2σ core fit, which the √(χ²/ndf) inflation then absorbs. Fit-parameter errors: PDG √(χ²/ndf) scaling when >1, implemented identically in timingFitSummary.C:68, systematicsPostfix.C:121, methodGainPostfix.C:147, depthDial.C:102, and cited (\cite{pdg}, radical_timing.tex:509). Headline check: 25.7/√2000 = 0.57 → the published ±0.6; DSB1 four-build fit χ²/ndf = 3.5/4 < 1 → no inflation, so a=203±6, b=18.8±0.8 are raw Minuit errors on σ/√2000 points. Consistent. FitGaussCore's own sigmaErr is only load-bearing in positionReconcile's printout (gausErr, positionReconcile.C:50) — where it prints ±0.00 at N=122k, which is why the house rightly prefers √(2N).

== 3. Paired bootstrap (methodGainPostfix.C) ==
Seed: TRandom3(20260610) fixed (line 97); killshot uses 20260609 (mixedKillshotBootstrap.C:126) — deterministic, stated in ANALYSIS_GUIDE.md:73. Resampling unit: per-event Poisson(1) weights, the SAME weight vector applied to all sources per replica (lines 118-121) — correct paired design; pairing cancels shower-population fluctuations as the AUDIT pre-registers. Windows frozen from the original sample per source (line 115) — well-defined statistic under resampling, at the cost of not propagating window uncertainty (second-order). Convention concerns: (a) total = RMS over 8 SIGNED shifts (lines 99-101). For the three two-sided knobs this averages up/down rather than taking per-knob envelopes; recomputing DSB1 with half-spread-per-knob quadrature from the committed shifts gives ~±1.9 ps vs the quoted ±1.0 — a ×2 convention sensitivity that is disclosed in the caption ('the total is the RMS of the shifts', tab_systematics.tex) but never argued anywhere.
